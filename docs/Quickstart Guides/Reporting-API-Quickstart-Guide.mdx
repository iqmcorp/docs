## About IQM Reports

The IQM APIs provide access to all dimensions and KPIs of your ad-serving Reporting data in JSON format. You can use the APIs to connect to applications of your choice.

Reports can be either **daily** or **aggregated**. The daily Report will include the date as a dimension and provide the data breakdown by date. Reports provide five top-level dimensions of filtering support, and you can filter on multiple values for each dimension.

You can run a Report containing up to three months of data for any dimension combination or up to one year for the Campaign dimension. You must run multiple Reports if you need more than that amount of data.

## Run a Report Using the IQM API

This QuickstartGuide will cover how to run a basic IQM Report.  

The minimum requirements to perform this task are: logging in with authentication credentials, selecting dimensions and metrics, choosing a timezone, and executing a Report. Once these steps are accomplished, more can be learned about IQM's API through the [Guidelines](/Guidelines/) pages.

1. Log In
      * Optional if you have already logged in and have a token
1. Request Dimensions and Metrics
      * Optional if you know supported/desired dimensions/metrics
1. Request Timezones
      * Optional if you already know supported/desired ID values
1. Execute the Report
      * Execute the Report with parameters from the previous steps

### Step 4: Execute a Report

<span class="badge badge--success">POST</span> <span class="path-text">/api/v3/ra/report/execute</span>

<div class="container">
  <div class="child1">

Run a Report based on Report data.

The Report will default to the previous three months if a start and end date are not included.

All columns are optional, but some must be included to run a Report.

At least one dimension is required.

For further information see the complete [Execute Report Data API](/Guidelines/Reports-API-Guidelines#execute-a-report).

| Headers |  |
| ----  | --- |
| `X-IAA-API-TOKEN` <br /><span class="type-text">string</span> <span class="required-text">required</span> | Advertiser API Token |


| Request Schema  |  |
| ---- | --- |
| `startDate` <br /><span class="type-text">integer</span> | Unix epoch timestamp of start date, in milliseconds |
| `endDate` <br /><span class="type-text">integer</span> | Unix epoch timestamp of end date, in milliseconds | 
| `dimensions` <br /><span class="type-text">string</span> | List of dimensions for the Report obtained from dimensions and metrics details API |
| `customerIds` <br /><span class="type-text">array of integers</span> | List of Customers IDs to include in the Report, leave empty array [ ] to include all, optional for workspace users only |
| `columns` <br /><span class="type-text">array of strings</span> | List of metrics for the Report obtained from dimensions and metrics details API | 
| `timezoneId` <br /><span class="type-text">integer</span> | Timezone ID |
| `requestType` <br /><span class="type-text">integer</span>| ID to identify if the Report request is Daily (<var>1</var>) or Aggregated (<var>2</var>). |
| `reportAggregated` <br /><span class="type-text">string</span> | Field to identify if the first Report dimension is Aggregated (<var>1</var>) or not (<var>0</var>) |
|`timezoneName` <br /><span class="type-text">string</span> | Timezone name |
| `filters` | object | Filter values to filter dimensions |
| `pageNo` <br /><span class="type-text">integer</span> | Page number for the data |
| `noOfEntries` <br /><span class="type-text">integer</span> | Maximum number of entries returned |
| `sortBy` <br /><span class="type-text">string</span> | Sorts by ascending (+) or descending (-), example: <var>-impressions</var> |
