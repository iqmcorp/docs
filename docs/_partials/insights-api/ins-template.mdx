### Get List of Templates

<CopyUrl method="GET" url="https://api.iqm.com/api/v3/ins/templates" />

<div class="endpoint-container">
  <div class="child1">

Get a list of all templates for the specified report type with optional filtering and pagination.

A <var>templateId</var> can be used to get [Get Template Details](#get-template-details) or to update an existing template using the [Update Report Template](#update-report-template) endpoint. 

| Query Parameters |  |
| ---- | --- |
| `category` <br /><span class="type-text">string</span> <span class="required-text">required</span> | Insight type category: <var>pld</var>, <var>ict</var>, or <var>nld</var> |
| `searchField` <br /><span class="type-text">string</span> | Filter templates by name |
| `noOfEntries` <br /><span class="type-text">integer</span> | Number of entries per page, default: <var>20</var> |
| `pageNo` <br /><span class="type-text">integer</span> | Page number for pagination, default: <var>1</var> |
| `frequencyIds` <br /><span class="type-text">integer</span> | Filter by [Scheduling Frequency IDs](/guidelines/insights-api#get-scheduling-frequencies) |
| `statusIds` <br /><span class="type-text">integer</span> | Filter by [Template Status IDs](/guidelines/insights-api#get-list-of-template-statuses) |
| `sortBy` <br /><span class="type-text">string</span> | Sorts by ascending (<var>+</var>) or descending (<var>-</var>), default: <var>-templateId</var> |

### Update Report Template

<CopyUrl method="PATCH" url="https://api.iqm.com/api/v3/ins/templates/{templateId}" />

<div class="endpoint-container">
  <div class="child1">

Apply a partial update to an existing report template. Only the fields present in the request are evaluated and applied.

The platform automatically re-evaluates future executions based on the effective configuration after the update. Past executions and generated reports are never modified.

:::warning[Important Notes]
- Updates only affect **future executions**
- A resumed template may not necessarily produce a future run
- "One Time" templates (frequency ID <var>1</var>) execute at most once
- If no future execution exists, nextRunDate will be null
:::


| Path Parameters |  |
| ---- | --- |
| `templateId` <br /><span class="type-text">integer</span> | The ID of the template to update |
