### PLD Eligibility Requirements

PLD Insights reports are available for political Campaigns that meet certain requirements. 

Read more about these requirements in our [Help Center<ExternalLink />](https://help.iqm.com/en/articles/10774817-provider-level-data-pld-insights-report-overview#h_81d3542441) article.

<table className="requirementsTable">
<th>Requirement Type</th>
<th>Requirement Details</th>
<th>Notes</th>
<tr>
  <td>Campaign vertical</td>
  <td>Healthcare</td>
  <td>Any Campaign created under a healthcare advertiser account is a “Healthcare” campaign.</td>
</tr>
<tr>
  <td>Campaign status</td>
  <td>Running, Expired (within the past 90 days), Paused (within the past 90 days)</td>
</tr>
<tr>
  <td>Campaign duration</td>
  <td>7 or more days</td>
</tr>
<tr>
  <td>Campaign limit</td>
  <td>Up to 50 eligible Campaigns from the same insertion order (IO) per report</td>
</tr>
<tr>
  <td>Audience type</td>
  <td>Matched Audience created from NPI IDs, Matched Audience created from raw data <br /><br /> Patient Matched Audiences are not eligible for PLD Insights reporting</td>
</tr>
</table>

### Get PLD Reports Interval

<CopyUrl method="GET" url="https://api.iqm.com/api/v3/ins/pld-reports/interval" />

<div className="endpoint-container">
  <div className="child1">

Get PLD reporting interval details for a specific template ID and trigger time.

To see more details about your PLD report, use the [Get Template Details](#get-template-details) endpoint. 

| Query Parameters |  |
| ---- | --- |
| `templateId` <br /><span class="type-text">integer</span> | [Template ID](#get-list-of-pld-templates) for which the reporting interval is to be fetched |
| `triggerTimeEpoch` <br /><span class="type-text">integer</span> | Unix epoch timestamp (in milliseconds) representing the trigger time |

<details className="objectPropertiesDetails" style={{ maxWidth: "800px", padding: "1rem" }}>

See [TypeScript Prerequisites](/getting-started/typescript-prerequisites.mdx) for usage.

```ts
import {
  getInstance
} from "prerequisites"

const axios = getInstance();

interface Responses {
  200: {
    content: {
      "application/json": {
        success: boolean;
        data: {
          startDateEpoch: number;
          endDateEpoch: number;
          nextRunDateEpoch: number;
        }
      }
    };
  };
}

function getReportInterval(): Promise<Responses> {
  const options = {
    method: 'GET',
    url: 'https://api.iqm.com/api/v3/ins/pld-reports/interval',
    params: {
      query: {
        templateId: `number`,
        triggerTimeEpoch: `number`,
      }
    }
  };

  return axios.request(options).then(({ data }: { data: Responses }) => data);
}
```
