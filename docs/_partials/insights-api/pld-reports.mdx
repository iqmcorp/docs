### PLD Eligibility Requirements

PLD Insights reports are available for political Campaigns that meet certain requirements. 

Read more about these requirements in our [Help Center<ExternalLink />](https://help.iqm.com/en/articles/10774817-provider-level-data-pld-insights-report-overview#h_81d3542441) article.

<table className="requirementsTable">
<th>Requirement Type</th>
<th>Requirement Details</th>
<th>Notes</th>
<tr>
  <td>Campaign vertical</td>
  <td>Healthcare</td>
  <td>Any Campaign created under a healthcare advertiser account is a “Healthcare” campaign.</td>
</tr>
<tr>
  <td>Campaign status</td>
  <td>Running, Expired (within the past 90 days), Paused (within the past 90 days)</td>
</tr>
<tr>
  <td>Campaign duration</td>
  <td>7 or more days</td>
</tr>
<tr>
  <td>Campaign limit</td>
  <td>Up to 50 eligible Campaigns from the same insertion order (IO) per report</td>
</tr>
<tr>
  <td>Audience type</td>
  <td>Matched Audience created from NPI IDs, Matched Audience created from raw data <br /><br /> Patient Matched Audiences are not eligible for PLD Insights reporting</td>
</tr>
</table>

### Create PLD Report Template

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/ins/pld-report-templates" />

<div class="endpoint-container">
  <div class="child1">

Creates a PLD report template for the given Campaigns and scheduling configuration.

The request defines a template name, associated Campaign IDs, frequency and frequency details (daily, weekly, monthly, or custom), template start/end dates, aggregation flag, recipient emails, and template status.

### Update PLD Report Template

<CopyUrl method="PATCH" url="https://api.iqm.com/api/v3/ins/pld-report-templates/{templateId}" />

<div class="endpoint-container">
  <div class="child1">

Apply a partial update to an existing PLD report template. Only the fields present in the request are evaluated and applied.

The platform automatically re-evaluates future executions based on the effective configuration after the update. Past executions and generated reports are never modified.

:::warning[Important Notes]
- Updates only affect **future executions**
- A resumed template may not necessarily produce a future run
- "One Time" templates (frequency ID <var>1</var>) execute at most once
- If no future execution exists, nextRunDate will be null
:::


| Path Parameters |  |
| ---- | --- |
| `templateId` <br /><span class="type-text">integer</span> | The ID of the PLD template to update |
