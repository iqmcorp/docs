# MCP Audience Tutorial

Use the IQM MCP to interact with the Audience API.

## Overview

This tutorial will guide you through using the IQM MCP to access and manage Audiences with the Audience API. You will learn how to upload a Matched Audience and create a Segmented Audience using the MCP interface. 

The Audience MCP supports all Audience operations as documented in the [Audience API](/guidelines/audience-api) Guidelines, so you can manage all types of Audiences using the MCP.

## Before You Begin

To create an Audience using the IQM MCP the following are required:

1. An Account On the IQM Platform
   *  See [Getting Started](/getting-started/before-you-begin) section to create an account and request a Client ID and Client Secret
1. CSV or XLSX File for Matching
   *  Download a <a href={useBaseUrl('/assets/political-matched-audience-example.csv')} download>Sample File</a>

<div style={{ maxWidth: "1000px" }}>
:::info 

**File requirements for Audience matching:**
  * CSV or XLSX file for Audience matching with maximum 1GB or 5M records
  * Without Voter ID / Email / Phone / Address in the file
    * One of the following system field combinations is required for matching
      * First Name, Last Name, Zip, State
      * Last Name, Street Address, Zip, State
      * Street Address, Zip, state
    * Other fields are optional and can be added along with the above fields
  * With Voter ID / Email / Phone / Address in the file
    * Only one of these fields is required - Voter ID or Email or Phone, or Address. The preferred format for full address is Street, City, Zip, and State
    * Other fields are optional and can be added along with one of the above fields

:::
</div>

## About IQM Audiences

The Audience API allows you to create different types of Audiences for Campaign targeting based on desired specifications. You can create new Audiences, retrieve details about existing Audiences, and update or delete Audiences as needed.
