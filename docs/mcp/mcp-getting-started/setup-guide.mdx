# Setup Guide

## Overview 

IQM MCP is a powerful interface that allows AI models to access information and functionality of IQM's various apps. Use this set up guide to get up and running with IQM MCP. 

## Before You Begin

Before you can use the IQM MCP, you will need an account on the [IQM Platform](https://app.iqm.com/). If you do not have an account, reach out to IQM Support (support@iqm.com) or your IQM representative to get started.

You will also need to have either [Claude Desktop](https://www.claude.com/download) or [GitHub Copilot](https://github.com/features/copilot) installed on your computer.

### Authorization Credentials 

You will need the following credentials for the MCP to connect to your IQM account:
* Organization Workspace ID
* Bearer Token

##### OW ID 

To find your Organization Workspace ID (OW ID):
1. Log in to the [IQM Platform](https://app.iqm.com/)
2. Click on your profile icon in the top right corner
3. Select "Manage my profile"
4. Navigate to "Organizations" tab
5. Your OW ID is listed on the User's Org card 

<center><img className='diagram' src={MCPowid} /></center>

##### Bearer Token 

To find your Bearer Token:
1. While logged in to the [IQM Platform](https://app.iqm.com/), right click anywhere on the page and select "Inspect" to open the Developer Tools.
2. Navigate to the "Application" tab in the Developer Tools.
3. In the left sidebar, under "Storage", click on "Local Storage" and then select the URL for the IQM Platform (https://app.iqm.com).
4. Look for the key named "auth" in the list of local storage items.
5. The value associated with the <var>access_token</var> is your Bearer Token. Copy this value for use in the MCP setup.

<center><img className='diagram' src={MCPBearer} /></center>

### Install MCP 

The installer contains an executable file to install the IQM MCP. You will be prompted to set up with Claude Desktop or GitHub Copilot. The installer will set up the config files and download the necessary specifications to get started with your chosen AI model. You will also be prompted for your [Authorization Credentials](#authorization-credentials).

<Tabs>
<TabItem value="macos" label="macOS" className='genericTab'>
  
  To install IQM MCP on macOS:
  1. Double-click the downloaded .zip file to extract installer contents.
  2. Open the <var>IQM MCP Installer</var> folder and Double-click <var>Installation Setup.workflow</var>.
    <center><img style={{ width:"50%"}} className='diagram' src={MCPInstSetup} /></center>
  3. Right-click <var>Install IQM MCP.command</var> and select `Quick Actions > Make Executable`.
    <center><img style={{ width:"50%"}} className='diagram' src={MCPExe} /></center>
  4. Double-click <var>Install IQM MCP.command</var> and follow the on-screen instructions to complete the installation.

  :::warning[NOTE]
  If you encounter a security warning when trying to run the installer, you may need to adjust your macOS security settings to allow the installation of applications from unidentified developers. 
  
  To do this, go to `System Settings > Privacy & Security > Security`, and under the "Allow apps downloaded from" section, select "Open Anyway". You may need to click the lock icon and enter your administrator password to make changes.

  <center><img style={{ width:"65%"}} className='diagram' src={MCPInstallerSecurity} /></center>
  :::

:::tip[Update Specifications or Credentials]

To get the latest IQM MCP specifications or to update your authorization credentials, run the <var>.command</var> executable again.

:::

### Github Copilot Setup

You may have to enable MCP tools in GitHub Copilot settings:
1. Click the `Configure Tools` icon in the Copilot Chat window.
 <center><img style={{ width:"65%"}} className='diagram' src={MCPChatTools} /></center>
2. In the `Tools panel`, check the <var>iqm-mcp-...</var> tools to enable them.
3. If you have updated the MCP specifications, you may need to click the `Update Tools` button to see the latest tools.
 <center><img style={{ width:"65%"}} className='diagram' src={MCPConfigTools} /></center>
