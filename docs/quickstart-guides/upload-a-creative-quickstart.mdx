# Upload a Creative

Upload your first Creative using IQM's API in this step-by-step Quickstart Guide.

The following endpoints will be used to upload a Creative:

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/ua/oauth/token" />
<br /><CopyUrl method="GET" url="https://api.iqm.com/api/v3/master/creativeTypes" />
<br /><CopyUrl method="POST" url="https://api.iqm.com/api/v3/crt/creatives" />
<br /><CopyUrl method="GET" url="https://api.iqm.com/api/v3/crt/creatives/\{creativeId\}" />


## About IQM Creatives

The IQM APIs provide access to upload a Creative in Image, Video, Audio, HTML, XLSX, or CSV formats. The APIs can be used to connect to the desired applications. 

### More Resources
  * [Creatives and Specifications<ExternalLink />](https://help.iqm.com/en/collections/1387054-creatives-and-specifications) Help Center Article
  * [Creative API](/guidelines/creative-api) Guidelines

## Before You Begin

To upload a Creative and create a Campaign, the following are required:

1. An Account On the IQM Platform
   *  See [Getting Started](/getting-started/before-you-begin) section to create an account and request a Client ID and Client Secret
1. Image, Video, Audio, HTML, VAST, DAAST, CSV, or XLSX File to Upload Creative

<div style={{ maxWidth: "1000px" }}>
:::info 

**File requirements for Creative file:**
  * 2MB for JPG, JPEG, PNG
  * 750KB for GIF
  * 400MB for MOV, MP4
  * 100MB for MP3, WAV, OGG, MPEG

For more information on [VAST<ExternalLink />](https://www.iab.com/guidelines/vast/) and DAAST XML format specifications, please see [IAB Tech Lab's Documentation<ExternalLink />](https://iabtechlab.com/standards/digital-audio-ad-serving-template-daast/).

Sample CSV file: [https://app.iqm.com/creatives/example-files/sample_html_creatives.csv<ExternalLink />](https://app.iqm.com/creatives/example-files/sample_html_creatives.csv)

:::
</div>

## Upload a Creative Using the IQM API

This Quickstart Guide will cover how to create a Campaign and upload a Creative. 

The minimum requirements to perform this task are: logging in with authentication credentials, uploading a Creative, and creating a Campaign. Once these steps are accomplished, more can be learned about IQM's API through the [Guidelines](/guidelines/) pages.

1. Log In
    * Optional if you have already logged in and have a token
1. Request Creative Types
    * Optional if you already know supported/desired types
1. Upload Creative File
1. Check Creative Status

### Step 3: Upload a Creative

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/crt/creatives" />

<div class="endpoint-container">
  <div class="child1">

Once you have the desired Creative Type ID, you can upload a Creative. With the two request parameters, <var>creativeRequest</var> and <var>creativeFiles</var>, you will add the Creative metadata and the Creative file respectively. 

<var>creativeRequest</var> maps an ID to fields defining the Creative metadata outlined in the table below and shown as an example in the Request Sample code block in the right-hand column. 

<var>creativeFiles</var> are added as multipart format. See the [MDN Documentation<ExternalLink />](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Methods/POST#multipart_form_submission) on Multipart/form-data encoding for more information.  

| Request Schema | |
| --- | --- | 
| `creativeRequest`<br /><span class="type-text">form data</span> <span class="required-text">required</span> | This parameter accepts a Map of IDs to Creative metadata. Each ID corresponds to a single Creative. |
| `creativeFiles`<br /><span class="type-text">multipart/form-data</span> | Array of Creative files to upload. Required only when Creative source type is <var>FILE</var> in the Creative metadata. Each uploaded file's name must exactly match the creativeSource value specified in the corresponding Creative object. Supports multiple file uploads for batch Creative creation. Accepted file types include images (JPEG, PNG, GIF), videos (MP4, WebM), and audio (MP3, WAV, OGG). |
