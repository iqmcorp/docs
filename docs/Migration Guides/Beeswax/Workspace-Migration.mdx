# Migrate a Workspace

Migrate a Workspace from Beeswax to IQM by comparing each platform's API methods for creating and managing Workspaces, Customers, and Advertisers.

The API Comparison table highlights the shared actions and API methods between IQM and Beeswax for creating and managing Workspaces, Customers, and Advertisers. Select an endpoint to review each platform's relevant API documentation. By comparing APIs, you can migrate your workflow from Beeswax to IQM. 

For a comprehensive overview of IQM's finance services, see the [Workspace API](/guidelines/workspace-api) Guidelines.

## Definitions

<center>
<table className='platformTable'>
<th>IQM</th>
<th>Beeswax</th>
<th>Description</th>
<tr>
  <td>Organization</td>
  <td>Account</td>
  <td>The entity containing an Organization's or company's details (address, email, website, etc.).</td>
</tr>
<tr>
  <td>Workspace (User)</td>
  <td>User</td>
  <td>A user that signs up with the platform and receives authentication credentials: Organization Workspace ID (<var>owId</var>) for IQM; Account ID for Beeswax. Workspace Users are also referred to as Advertisers or Customers in different contexts.</td>
</tr>
<tr>
  <td>Advertiser (Customer)</td>
  <td>Advertiser</td>
  <td>Once an Advertiser is assigned to a Workspace User, they become a Customer (they also operate as their own Workspaces). An Advertiser can be assigned to one Workspace; and they can have multiple Insertion Orders assigned to them.</td>
</tr>
</table></center>
