# Migrate an Audience

## Overview 

The API Comparison table highlights the shared actions and API methods between IQM and Beeswax for creating and managing Audiences. Select an endpoint to review each platform's relevant API documentation. By comparing APIs, you can migrate your workflow from Beeswax to IQM. 

For a comprehensive overview of IQM's Audience services, see the [Audience API](/Guidelines/Audience-API-Guidelines).

## Definitions

<center>
<table className='platformTable'>
<th>IQM</th>
<th>Beeswax</th>
<th>Description</th>
<tr>
  <td>Audience</td>
  <td>Targeting</td>
  <td>The group of people targeted for a Campaign. Audiences comprise included/excluded demographic data which are used to optimize targeting.</td>
</tr>
<tr>
  <td>Audience Segment</td>
  <td>Segment Targeting</td>
  <td>A group of specified Segments with optimization settings.</td>
</tr>
<tr>
  <td>Segment Group</td>
  <td>Targeting Modules and Keys</td>
  <td>The demographic characteristics by which a group of people can be identified, organized, and combined into Audience Segments (Segment Targeting for Beeswax).</td>
</tr>
<tr>
  <td>Matched Audience</td>
  <td>n/a</td>
  <td>A customized Audience that you've created and uploaded to the platform.</td>
</tr>
<tr>
  <td>Pre-bid Audience</td>
  <td>Content Module</td>
  <td>An Audience composed of segments that have been sent to the platform by third-party data providers that you can purchase and target in your Audiences.</td>
</tr>
<tr>
  <td>Geo-Farmed Audience</td>
  <td>Geo Module</td>
  <td>An Audience created based on a customizable geographic location rather than demographic data.</td>
</tr>
<tr>
  <td>Contextual Audience</td>
  <td>Content Module</td>
  <td>An Audience generated by keywords or URLs, based on user activity.</td>
</tr>
</table></center>
