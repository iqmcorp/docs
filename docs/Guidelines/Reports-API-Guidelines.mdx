# Reports API

The Reports API allows you to create and schedule custom Reports to generate Insights and inform current and future Campaign optimization.

Metrics and dimensions can be selected to customize Reports. Metrics are quantitative values (number, percentage, dollar amount) measuring some Campaign's attribute. Dimensions are those attributes: the chosen element (e.g. demographics, Creatives, technologies) to be subjected to a metric. 

Reports can be scheduled to a specified delivery frequency, and further fine-tuned to aggregated or disaggregated (daily) data. 


<div style={{ maxWidth: "1000px" }}>
:::info

There must be at least one active Campaign before a Report can be created/executed. 

See the following Quickstart Guides:
  * [Create a Campaign](/Quickstart%20Guides/Create-a-Campaign-Quickstart)
  * [Run a Report](/Quickstart%20Guides/Reporting-API-Quickstart-Guide)
  * [Schedule a Report](/Quickstart%20Guides/Schedule-Report-API-Quickstart-Guide)

:::

:::warning[NOTE]

The date range for a Report cannot exceed one year. 

:::
</div>

Learn more about Reports with our [Help Center<ExternalLink />](https://help.iqm.com/en/articles/10343665-reports-overview) articles.

### Execute a Report

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/ra/report/execute" />

<div class="container">
  <div class="child1">

Execute a report by ID or with new Report data. 

:::warning[NOTE]

The date range for a Report cannot exceed one year. 

:::

| Request Schema|  |
| ---- | --- |
| `id` <br /><span class="type-text">integer</span> | Executes Report by [Report ID](#get-a-list-of-reports) |
| `dimensions` <br /><span class="type-text">string</span> | Dimensions to filter by in serialized JSON format |
| `columns` <br /><span class="type-text">array of strings</span> | List of metrics for the Report obtained from dimensions and metrics details API |
| `customerIds` <br /><span class="type-text">array of integers</span> | [Customer ID](/Guidelines/Workspace-Guidelines#get-multi-level-customers-list)s to add to Report |
| `filters` <br /><span class="type-text">JSON string</span> | Supports <var>campaignId</var>, <var>campaignStatus</var>, and <var>creativeTypeId</var> filtering |
| `searchField` <br /><span class="type-text">string</span> | Filters by searched keyword |
| `requestType` <br /><span class="type-text">integer</span> | [Report Request Type ID](#get-report-request-types)
| `reportAggregated` <br /><span class="type-text">string</span> | ID for whether first Report dimension is "aggregated" (<var>1</var>) or "disaggregated" (daily) (<var>0</var>) |
| `startDate` <br /><span class="type-text">integer</span> | Unix epoch timestamp of start date for Report |
| `endDate` <br /><span class="type-text">integer</span> | Unix epoch timestamp of end date for Report |
| `timezoneId` <br /><span class="type-text">integer</span> | [Timezone ID](/Guidelines/Master-API-Guidelines#get-timezones) |
| `timezoneName` <br /><span class="type-text">string</span> | Timezone name |
| `sortBy` <br /><span class="type-text">string</span> | Sorts by ascending (<var>+</var>) or descending (<var>-</var>), default: <var>+impressions</var>
| `pageNo` <br /><span class="type-text">integer</span> | Page number for the data, default: <var>1</var> |
| `noOfEntries` <br /><span class="type-text">integer</span> | Maximum number of entries returned, default: <var>50</var> |
