# Creative API

## Overview

The Creative API allows users to upload images, videos, audio, or HTML files to their Creative library. The user get Creative details and manage Creatives, Creative groups and associated conversions. 

This page covers the common endpoints and methods associated with the Creative API.

Learn more about Creatives with our [Help Center<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-linecap="round" stroke-linejoin="round" width="1em" height="1.2em" stroke-width="1.5"><path d="M12 6h-6a2 2 0 0 0 -2 2v10a2 2 0 0 0 2 2h10a2 2 0 0 0 2 -2v-6"></path><path d="M11 13l9 -9"></path><path d="M15 4h5v5"></path></svg>](https://help.iqm.com/en/collections/1387054-manage-your-creatives) articles.

## Creative Management

### Add New Creative

<span class="badge badge--success">POST</span><span class="path-text">/api/v3/crt/creatives</span>

<div class="container">
  <div class="child1">

Add a new Creative by providing the Creative metadata and files. Endpoint accepts a JSON payload with Creative details and supports file uploads in multipart form data. The API handles both the metadata and Creative file uploads simultaneously. 

| Request Schema | |
| --- | --- | 
| `creativeRequest`<br /><span class="type-text">string</span> | This parameter accepts a Map of UUIDs to creative details. Each UUID corresponds to a single creative. The metadata includes information such as creative name, external creative ID, creative type, and click URL. Additionally, the creative files are sent as multipart form data |
| `creativeFiles`<br /><span class="type-text">array of strings</span> | Array of creative files uploaded as multipart files. These files will be used for creating the creatives. Each file is processed according to its type (image, video, etc.) |

</div>

### Update Creative Details

<span class="badge badge--info">PATCH</span><span class="path-text">/api/v3/crt/creatives/\{creativeId}</span>

<div class="container">
  <div class="child1">

Update the details of an existing Creative. Users authorized to modify a Creative's properties are: advertisers, workspaces, or super users. Ownership and edit permissions are validated to ensure request is authorized. 

| Path Parameters | |
| --- | --- |
| `creativeId` <br /><span class="type-text">integer</span> | Creative ID |

| Request Schema | |
| --- | --- |
| `creativeName` <br /><span class="type-text">string</span> | Creative name
| `externalCreativeId` <br /><span class="type-text">string</span> | 3rd party ID for Creative 
| `creativeRejectionReason` <br /><span class="type-text">string</span> | Reason for rejecting Creative if applicable
| `creativeRemarks`<br /><span class="type-text">string</span> | Adds remarks to Creative 

</div>
<div class="child2">

<Tabs>
<TabItem value="JSON" label="JSON" default>

```json title="Request Sample"
{
  "creativeName": "New creative name",
  "externalCreativeId": "3rd party id"
}
```

```json title="Response 200"
{
  "success": true,
```
