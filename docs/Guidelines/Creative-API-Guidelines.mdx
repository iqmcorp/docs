# Creative API

## Overview

The Creative API allows you to upload images, videos, audio, or HTML files to the Creative library. You can also get Creative details and manage Creatives, Creative groups and associated Conversions. This page covers the common endpoints and methods associated with the Creative API.

<div style={{ maxWidth: "1000px" }}>
:::info 

**File requirements for Creative file:**
  * 2MB for JPG, JPEG, PNG
  * 750KB for GIF
  * 400MB for MOV, MP4
  * 100MB for MP3, WAV, OGG, MPEG

For more information on [VAST<ExternalLink />](https://www.iab.com/guidelines/vast/) and DAAST XML format specifications, please see [IAB Tech Lab's Documentation<ExternalLink />](https://iabtechlab.com/standards/digital-audio-ad-serving-template-daast/).

Sample CSV file: [https://api.iqm.com/creatives/example-files/sample_html_creatives.csv<ExternalLink />](https://api.iqm.com/creatives/example-files/sample_html_creatives.csv)

:::
</div>

          };

            owId: number;
            organizationName: string
          }
        }
      };
    };
  };
}

function getCreative(): Promise<Responses> {
  const options = {
    method: 'GET',
    url: 'https://api.iqm.com/api/v3/crt/creatives/{creativeId}',
    params: {
      path: {
        creativeId: `number`
      }
    }
  };

  return axios.request(options).then(({ data }: { data: Responses }) => data);
}

### Get List of Creatives and Details

<span class="badge badge--success">POST</span><span class="path-text">/api/v3/crt/creatives/list</span>

<div class="container">
  <div class="child1">

Get a list of Creatives with their details. 

A Creative <var>id</var> can be used to [Get Creative Details by ID](#creative-details-by-id) and to perform various actions in [Creative Managment](#creative-management) endpoints.

| Request Schema | |
| --- | --- | 
| `pageNo`<br /><span class="type-text">integer</span> | Number of pages for retrieved data
| `noOfEntries`<br /><span class="type-text">integer</span> | Maximum number of entries per page
| `sortBy`<br /><span class="type-text">string</span> | Sorts by ascending (<var>+</var>) or descending (<var>-</var>) 
| `searchField`<br /><span class="type-text">string</span> | Search results by keyword
| `platformCreativeTypeIds`<br /><span class="type-text">array of integers</span> | [Creative Type IDs](/Guidelines/Creative-API-Guidelines#get-creative-types-list)
| `creativeStatusIds`<br /><span class="type-text">array of integers</span> | [Creative Status IDs](/Guidelines/Creative-API-Guidelines#get-creative-status-list)
| `creativeIds`<br /><span class="type-text">array of integers</span> | [Creative ID](#get-list-of-creatives-and-details)s
| `creativeGroupId`<br /><span class="type-text">integer</span> | Filter results by [Creative Group ID](#get-list-of-creative-groups)
| `campaignIds`<br /><span class="type-text">array of integers</span> | Filter results by [Campaign ID](/Guidelines/Campaign-API-Guidelines#get-list-of-campaigns)s
| `startTime`<br /><span class="type-text">integer</span> | Unix epoch timestamp (in milliseconds) of start time
| `endTime`<br /><span class="type-text">integer</span> | Unix epoch timestamp (in milliseconds) of end time
| `owIds`<br /><span class="type-text">integer</span> | (Super/Workspace only) Organization Workspace IDs 
| `allOwIds`<br /><span class="type-text">boolean</span> | (Super/Workspace only) Filters records with all allowed OW IDs if <var>true</var>, prioritizes records over <var>owIds</var>
| `includeCreativeSourceDetails`<br /><span class="type-text">boolean</span> | (Advertiser only) Flag indicating whether to include Creative source details (<var>true</var>) or not (<var>false</var>)
| `includeOrgnizationDetails`<br /><span class="type-text">boolean</span> | Flag indicating whether to include Organization details (<var>true</var>) or not (<var>false</var>)
| `includeUserDetails`<br /><span class="type-text">boolean</span> | Flag indicating whether to include user details (<var>true</var>) or not (<var>false</var>)

### Add New Creative

<span class="badge badge--success">POST</span><span class="path-text">/api/v3/crt/creatives</span>

<div class="container">
  <div class="child1">

Add a new Creative by providing the Creative metadata and files. Endpoint accepts a JSON payload with Creative details and supports file uploads in multipart form data. The API handles both the metadata and Creative file uploads simultaneously. 

Please refer to [MDN Documentation<ExternalLink />](https://developer.mozilla.org/en-US/Learn/Forms/Sending_and_retrieving_form_data) on form data format.

| Query Parameters | |
| --- | --- | 
| `creativeRequest`<br /><span class="type-text">string</span> | This parameter accepts a Map of UUIDs to Creative details. Each UUID corresponds to a single Creative. |
| `creativeFiles`<br /><span class="type-text">array of strings</span> | Array of Creative files uploaded as multipart files. These files will be used for creating the Creatives. Each file is processed according to its type (image, video, etc.) |
