# User Management API

With the User Management API you can perform registration operations such as signing up, logging in, and other profile configurations.

This page will go over the common endpoints associated with user management.

Learn more about User Profiles with our [Help Center<ExternalLink />](https://help.iqm.com/en/collections/14628352-account-setup-and-management) articles. 

### Authentication

Use the following header parameters for all requests:

  <div class="smallTable">

| Headers  |  |
| ----  | --- |
| `Authentication` <br /><span class="type-text">string</span> <span class="required-text">required</span> | Authentication bearer token or Client ID and Secret <br />See [Authentication Guide](/Quickstart%20Guides/Authentication-Quickstart-Guide) |
| `X-IAA-OW-ID` <br /><span class="type-text">integer</span> <span class="required-text">required</span> | Organization Workspace ID Header |

</div>

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/ua/oauth/token" />

<div class="container">
<div class="child1">

This endpoint functions as the main authenticator for external connections. 

This API accepts a URL encoded form with different grant types. This endpoint can be used to log in via user password, authorization code or client credentials, or it can be used to refresh an access token. 

The API will send an [OAuth Compliant Response<ExternalLink />](https://www.oauth.com/oauth2-servers/authorization/the-authorization-response/), as seen in the example <var>Response 200</var> on the right side column.

| Authentication |  |
| --- | --- |   
| `Authentication` <br /><span class="type-text">string</span> <span class="required-text">required</span> | Client ID and Client Secret |

| Request Parameters | |
| --- | --- |
| `grantType` <br /><span class="type-text">string</span> | [OAuth Grant Types<ExternalLink />](https://oauth.net/2/grant-types/): <br /><var>authorization_code</var>: log in with redirect URL and client ID and secret <br /><var>password</var>: log in with username and password <br /><var>refresh_token</var>: obtain a new access token|
| `refresh_token` <br /><span class="type-text">string</span> | Refresh token to obtain new access token |
| `email` <br /><span class="type-text">string</span> | User's email |
| `password` <br /><span class="type-text">string</span> | User's password |
| `client_id` <br /><span class="type-text">string</span> | Client ID |
| `client_secret` <br /><span class="type-text">string</span> | Client secret |


</div>
