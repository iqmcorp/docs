# Creative API

The Creative API allows you to upload images, videos, audio, or HTML files to the Creative library. 

You can also get Creative details and manage Creatives, Creative groups and associated Conversions. This page covers the common endpoints and methods associated with the Creative API.

<div style={{ maxWidth: "1000px" }}>
:::info 

**File requirements for Creative file:**
  * 2MB for JPG, JPEG, PNG
  * 750KB for GIF
  * 400MB for MOV, MP4
  * 100MB for MP3, WAV, OGG, MPEG

For more information on [VAST<ExternalLink />](https://www.iab.com/guidelines/vast/) and DAAST XML format specifications, please see [IAB Tech Lab's Documentation<ExternalLink />](https://iabtechlab.com/standards/digital-audio-ad-serving-template-daast/).

Sample CSV file: [https://app.iqm.com/creatives/example-files/sample_html_creatives.csv<ExternalLink />](https://app.iqm.com/creatives/example-files/sample_html_creatives.csv)

:::
</div>

            owId: number;

          }
        }
      };
    };
  };

function getCreative(): Promise<Responses> {
  const options = {
    method: 'GET',
    url: 'https://api.iqm.com/api/v3/crt/creatives/{creativeId}',
    params: {
      path: {
        creativeId: `number`
      }
    }
  };

  return axios.request(options).then(({ data }: { data: Responses }) => data);
}

### Get List of Creatives and Details

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/crt/creatives/list" />

<div class="endpoint-container">
  <div class="child1">

Get a list of Creatives with their details. 

A Creative <var>id</var> can be used to [Get Creative Details by ID](#creative-details-by-id) and to perform various actions in [Creative Managment](#creative-management) endpoints.

| Request Schema | |
| --- | --- | 
| `pageNo`<br /><span class="type-text">integer</span> | Page number of retrieved data
| `noOfEntries`<br /><span class="type-text">integer</span> | Number of entries returned per page
| `sortBy`<br /><span class="type-text">string</span> | Sorts by ascending (<var>+</var>) or descending (<var>-</var>) 
| `searchField`<br /><span class="type-text">string</span> | Filter results by search field
| `platformCreativeTypeIds`<br /><span class="type-text">array of integers</span> | [Creative Type IDs](/guidelines/creative-api#get-creative-types-list)
| `creativeStatusIds`<br /><span class="type-text">array of integers</span> | [Creative Status IDs](/guidelines/creative-api#get-creative-status-list)
| `creativeIds`<br /><span class="type-text">array of integers</span> | [Creative ID](#get-list-of-creatives-and-details)s
| `creativeGroupId`<br /><span class="type-text">integer</span> | Filter results by [Creative Group ID](#get-list-of-creative-groups)
| `campaignIds`<br /><span class="type-text">array of integers</span> | Filter results by [Campaign ID](/guidelines/campaign-api#get-list-of-campaigns)s
| `startTime`<br /><span class="type-text">integer</span> | Unix epoch timestamp (in milliseconds) of start time
| `endTime`<br /><span class="type-text">integer</span> | Unix epoch timestamp (in milliseconds) of end time
| `owIds`<br /><span class="type-text">integer</span> | (Super/Workspace only) Organization Workspace IDs 
| `allOwIds`<br /><span class="type-text">boolean</span> | (Super/Workspace only) Filters records with all allowed OW IDs if <var>true</var>, prioritizes records over <var>owIds</var>
| `includeCreativeSourceDetails`<br /><span class="type-text">boolean</span> | (Advertiser only) Flag indicating whether to include Creative source details (<var>true</var>) or not (<var>false</var>)
| `includeOrgnizationDetails`<br /><span class="type-text">boolean</span> | Flag indicating whether to include Organization details (<var>true</var>) or not (<var>false</var>)
| `includeUserDetails`<br /><span class="type-text">boolean</span> | Flag indicating whether to include user details (<var>true</var>) or not (<var>false</var>)

## Creative Management

### Add New Creative

<CopyUrl method="POST" url="https://api.iqm.com/api/v3/crt/creatives" />

<div class="endpoint-container">
  <div class="child1">

This API allows you to add Creatives of various types by providing the Creative metadata and Creative files using multipart/form-data. The metadata is sent as a JSON string in the <var>creativeRequest</var> form field, and files (when creativeSourceType is <var>FILE</var>) are uploaded in the <var>creativeFiles</var> form field. The API supports batch creation of multiple Creatives in a single request, with partial success/failure handling.

Please refer to [MDN Documentation<ExternalLink />](https://developer.mozilla.org/en-US/Learn/Forms/Sending_and_retrieving_form_data) on form data format.

| Request Schema | |
| --- | --- | 
| `creativeRequest`<br /><span class="type-text">form data</span> <span class="required-text">required</span> | This parameter accepts a Map of IDs to Creative metadata. Each ID corresponds to a single Creative. |
| `creativeFiles`<br /><span class="type-text">multipart/form-data</span> | Array of Creative files to upload. Required only when Creative source type is <var>FILE</var> in the Creative metadata. Each uploaded file's name must exactly match the creativeSource value specified in the corresponding Creative object. Supports multiple file uploads for batch Creative creation. Accepted file types include images (JPEG, PNG, GIF), videos (MP4, WebM), and audio (MP3, WAV, OGG). |
